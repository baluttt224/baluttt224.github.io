Magic Square Project

CISC 131

This project asks the users for an odd integer greater than two. If the number entered fits the criteria, the program creates a magic square according to the user's specified height and width. As defined by Wikipedia, "In recreational mathematics, a magic square is an array of distinct positive integers such that the sums of the integers in each row, column and main diagonal are equal. The integer is the order of the magic square and the constant sum is called the magic constant."

Assembler Project

CISC 340

In this assembly-language translation, we mainly divided the project into three parts. The first part consisted of creating an array of labels (or recognizing that an address didn’t have a label). The second part consisted of converting each line of assembly-language to a decimal number. The third part consisted of writing the decimal numbers to the output file. In the first part, we recorded two command line inputs that the user typed, which needed an input file name and output file name. Then, we opened the input file and used a while loop to read through every line of the input file in order to get all of the labels (which we stored in a two-dimensional array of characters). If a line didn’t have a label, we put a “\0” for that corresponding spot of the label array. We stored the total number of lines in an input file. Also, we also did error checking. This consisted of labels not starting with letters, labels containing weird characters, labels with more than six characters, and using one of the instructions as label. After we got the label array, we checked to see if any duplicate labels existed. In the second part, we use strtok() to get each column of the current line of assembly-language. Then, we checked the first column to see if it was an instruction or not. After that, we checked for an error in which the label started with white space. Then, we used a series of if/else statements to determine the instruction type, the assembler directive, etc. We also checked for unknown instructions (which we obviously can’t compute). Inside each if/else statement, we used strtok() to get the rest of the columns. The usage count depended on the instruction type. We used strtok() three extra times for I-type and R type instructions, two extra times for J-type instructions, one extra time for assembler directives, and no extra times for O-type instructions. Then, we combined the columns of numbers and/or labels to a 32 bit binary, converted them to a long and stored in an output array. Lastly, we wrote the output array to the output file name.
